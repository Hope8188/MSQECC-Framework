# CSR Failure What-Ifs

## Purpose
This document asks the five failure questions the program has been avoiding and answers them directly.

## What we likely did wrong so far
The shortest honest list is:
- we treated a target condition like `q = 3` as if it were already close to a derivation
- we searched branch families before fully typing the underlying theory objects
- we let aggregate `R^2` speak louder than per-galaxy shape failure for too long
- we tried to solve closure before deriving normalization and transition scale together
- we exported results before exporting the theory object that would let others reproduce the logic mechanically

## Five what-if questions

### 1. What if the actual `T^3` machinery never produces the fixed-point condition we need?
Then the CSR galaxy-law route dies cleanly.

That would mean:
- the bridge result may still matter as a quantum-information or open-systems result
- the lattice geometry may still matter as a mathematical backbone
- but the galaxy-law claim would not follow from them

This is the cleanest failure mode because it is falsifiable.

### 2. What if `q = 3` or an equivalent deep-IR scaling does emerge, but the transition scale is still free?
Then CSR has not derived a law.
It has only rebuilt MOND-like asymptotics with a fitted constant.

That would still be useful negative information:
- the backbone may constrain shape
- but it would not explain why the bend occurs at the observed acceleration scale

In that case the program would have shown structural relevance, not physical completion.

### 3. What if the transition scale emerges, but quotient normalization stays underived?
Then the theory may get the bend at the right place and still miss galaxy shape.

That is close to what the current record already hints at:
- envelope behavior can look right
- radial residuals can still stay highly correlated

This failure would mean the hard missing object is not just the scale.
It is the map that turns the internal theory into the right radial response.

### 4. What if the whole mistake is the scalar reduction itself?
Then all current branches are projections of the wrong observable.

A real CSR law could then be:
- nonlocal rather than local
- multi-field rather than single-field
- tensorial rather than scalar
- filtered/measure-driven rather than expressible as one closed interpolation function

If this is true, the present branch family was never going to close the problem because it was compressing the wrong object too early.

### 5. What if the route fails because the program is underdefined, not because the physics is false?
Then the failure is partly procedural.

That would mean:
- the backbone may contain real content
- but the project has not made that content externally legible enough to test
- and outside models will keep returning MOND restatements or refusals because the task is not yet typed tightly enough

This is why the typed-object document matters.

## If someone derived actual CSR, what would it look like?
It would not look like the present template-plus-closure workflow.
It would look like this:

1. A fully explicit `T^3` action or generator.
2. A fully explicit quotient and measure.
3. A coarse-graining or RG map with a mechanically computable IR object.
4. A theorem or controlled numerical result showing the deep-IR fixed point.
5. A derivation of both:
   - the transition scale
   - the quotient normalization
6. A derived observable map to `g(r)` or `v(r)`.
7. Only then, galaxy benchmarking.

That is what “actual CSR” would look like.

## Why that is different from ours
The current program has:
- strong target diagnostics
- strong falsification infrastructure
- a good envelope template
- multiple failed closure attempts
- no live branch

What it does not yet have is the thing a real derivation would start with:
- a self-contained typed theory object whose IR limit can be computed by someone other than the original author

So the difference is not mostly numerical.
It is architectural.

## The best single picture
The current program looks like this:
- a clear mountain peak is visible
- we have decent instruments
- but we have not yet drawn the map of the mountain

A real derivation would invert that:
- first the map
- then the climb
- then the summit check

Right now we have spent too much effort checking whether different paths feel uphill without first proving they lie on the same mountain.

## Best remaining path after cutting everything else
After cutting every weak branch and every soft claim, the best remaining path is:
- write the typed `T^3` theory object
- compute the fixed-point quantity or anomalous dimension from that object
- stop if it misses the target
- continue only if it yields the deep-IR scaling and opens a route to transition-scale derivation

That is the highest-value surviving line.

## Bottom line
If CSR fails no matter what, the most likely reason is not that we lacked one more fit family.
It is that we tried to derive galaxy phenomenology from an under-specified theory object.

If CSR succeeds, it will do so by looking much more formal and much less heuristic than the program has looked so far.
