# MSQECC T3 Foundation Audit

**Date:** March 8, 2026

## Direct Verdict

If MSQECC is forced to use one code manifold and one mathematical core, the most defensible choice is the `Z2` toric code on `T^3`.

Once that choice is made, five things become clear:

1. the stabilizer algebra is explicit and standard,
2. the code encodes `3` logical qubits on `T^3`,
3. the plain toric code does not define separate gas and stellar sectors,
4. the plain toric code therefore does not derive `k = 3*pi`,
5. the plain toric code does not naturally produce an inverse-square idle-noise amplitude or the polynomial `4*pi^3 + pi^2 + pi`.

That means the present MSQECC interpretation is not yet a finished theory. It is a galaxy phenomenology layer sitting on top of a topological code vocabulary that has not been connected to matter strongly enough to generate the advertised predictions.

## 1. Stage 1.1: Choose One Code Manifold

The repository alternated between `S^3`, `T^3`, and unspecified compact 3-manifolds.

For the foundation work, that ambiguity has to stop.

The cleanest mathematical choice is `T^3`, the 3-torus, because:

- a periodic cubic lattice realizes it directly,
- the toric-code stabilizer algebra is standard on periodic lattices,
- the encoded-qubit count is controlled by homology,
- and `b_1(T^3) = 3` is at least compatible with the recurring three-cycle narrative.

This is a choice of mathematical convenience and consistency. It is not yet a proof that the universe uniquely selects `T^3` over the other compact flat 3-manifolds.

## 1.9 Core Quotient and Measure (Minimal Core Definition)

This section promotes two minimal objects to the core layer, because they are standard for the toric-code backbone and do not insert any galaxy-law asymptotics.

Quotient (core):
- Let H be the Hilbert space of edge qubits.
- Let S be the stabilizer group generated by vertex and plaquette checks.
- Define the physical code space as the +1 eigenspace of S.
- Define the quotient Q = H / S as the space of equivalence classes under the stabilizer action.

Measure (core):
- Define a normalized trace measure on the code space: mu(O) = Tr_code(O) / dim(H_code).
- This sets normalization only; it does not encode MOND or any IR scaling.

## 2. Stage 1.2: Write the Stabilizer Algebra Explicitly

Take an `L x L x L` periodic cubic lattice.

- Qubits live on edges.
- The Hilbert space is `H = (C^2)^(tensor E)` with `E = 3L^3` edge qubits.
- For each vertex `v`, define the star operator
  - `A_v = product_{e incident on v} X_e`
- For each plaquette `p`, define the flux operator
  - `B_p = product_{e in boundary(p)} Z_e`
- The Hamiltonian is
  - `H = - sum_v A_v - sum_p B_p`

Commutation structure:

- Every `A_v` commutes with every `A_v'`.
- Every `B_p` commutes with every `B_p'`.
- `A_v` and `B_p` overlap on either `0` or `2` edges of the cubic lattice, so they commute.

Code space:

- The code space is the simultaneous `+1` eigenspace of all stabilizers.
- Logical operators are noncontractible Wilson operators on `T^3`.

A convenient logical basis is:

- electric loops `W_x^e`, `W_y^e`, `W_z^e` made from `Z` along the three noncontractible 1-cycles,
- magnetic membranes `W_x^m`, `W_y^m`, `W_z^m` made from `X` over the dual noncontractible 2-cycles.

Each loop-membrane pair intersects once and anticommutes; pairs with different directions commute. This gives `3` encoded qubits.

## 4. Stage 1.3: Define Gas and Stellar Sectors in the Code

This is where the current framework breaks.

The plain `T^3` toric code distinguishes:

- electric point defects,
- magnetic flux loops,
- logical Wilson loops and membranes,
- and their stabilizer sectors.

It does **not** distinguish:

- neutral HI gas from stars,
- coherent baryonic gas from decohered stellar populations,
- or any matter-sector split that would give two different entanglement weights for the same underlying edge-qubit code.

All edge qubits are equivalent under the bare stabilizer algebra. The code supplies topology, not baryonic taxonomy.

To derive a gas-versus-stellar weight, MSQECC would need an extra structure that does not currently exist in the repository:

- an explicit matter Hilbert space,
- a coupling map from baryonic phases into code operators,
- and a rule showing why gas occupies one syndrome class while stars occupy another.

Without that map, there is no mathematically defined "gas sector" and no mathematically defined "stellar sector" inside the toric code.

## 5. Stage 1.4: Compute k

Once Stage 1.3 fails, the `k` derivation fails with it.

Still, it is useful to ask whether any obvious toric-code invariant lands at `3*pi`.

Natural quantities in the plain `T^3` toric code are:

- vertex valence `6`,
- plaquette boundary length `4`,
- logical-qubit count `3`,
- code rate `K/N = 3 / (3L^3) = 1 / L^3`,
- integer homology ranks over `GF(2)`.

None of these yield `3*pi`.

The number `pi` does not arise from the stabilizer algebra itself. It enters only when one imposes a continuum geometric parameterization of the torus, for example using angles in `[0, 2*pi)` or radii `R_i` with cycle lengths `2*pi R_i`. That is geometry placed on top of the code, not a stabilizer invariant.

Therefore the corrected conclusion is:

- the standard toric code on `T^3` does **not** derive `k = 3*pi`.
- the current value of `k` remains phenomenological.

## 6. Stage 2: Quantum-Noise Amplitude

### 6.1 `T^ent` Is Not Yet an Operator

The repository talks about an entanglement stress tensor `T^ent_{mu nu}` as if it were already defined in the code Hilbert space.

It is not.

There is no explicit operator formula in the local MSQECC materials giving:

- which qubits it acts on,
- which stabilizer combinations generate it,
- what its matrix elements are,
- or what scale sets its magnitude.

Without that, the hardware claim never had a quantitative source.

### 6.2 What the Plain Toric Code Predicts Instead

In the plain gapped local `Z2` toric code:

- ground-state connected correlations of separated local check observables are short-ranged and, in the stabilizer limit, collapse to zero beyond local overlap,
- finite-temperature excitations are defect strings and membranes, giving local or thermally screened behavior rather than a universal `1/D^2` tail.

That means the bare toric code does **not** naturally predict the IBM signature that was claimed.

The hardware results are therefore not just a failed measurement campaign. They are evidence that the advertised inverse-square law is not a consequence of the standard `T^3` toric-code foundation.

### 6.3 Amplitude Conclusion

For the plain toric code, the long-range inverse-square amplitude is effectively absent.

So the correct amplitude statement is:

- if MSQECC uses only the standard `T^3` toric code, the predicted long-range idle-correlation amplitude is `0` at the stabilizer level.
- to obtain a nonzero `1/D^2` law, the framework would need a new mediator beyond the standard commuting-projector model.

## 7. Stage 3: The alpha Problem

The coincidence

`4*pi^3 + pi^2 + pi = 137.036303776`

is numerically striking.

But the plain `T^3` toric code does not contain the ingredients required to derive the fine-structure constant:

- it is a `Z2` code, not a `U(1)` gauge sector,
- its algebra is combinatorial over `GF(2)`,
- and its natural invariants are integers, parities, and homology ranks.

### 7.1 Where pi Actually Appears

In this foundation choice, `pi` enters only through continuum conventions layered onto the torus:

- cycle lengths `2*pi R_i`,
- continuum Fourier integrals over angular coordinates,
- and geometric volume factors if one embeds the discrete code in a continuous manifold.

Those are real sources of `pi`, but they do not single out the polynomial `4*pi^3 + pi^2 + pi`.

### 7.2 Foundation Conclusion for alpha

The plain `T^3` toric code does not derive `alpha` and does not explain the specific polynomial in `pi`.

So the current status of the alpha formula is:

- extraordinary coincidence,
- useful theoretical lead,
- not yet a derivation.

## 8. Stage 4: Cosmological Constant and Code Rate

For the `T^3` toric code on an `L x L x L` periodic cubic lattice:

- physical qubits `N = 3L^3`,
- logical qubits `K = 3`,
- code rate `K/N = 1/L^3`.

That is the actual code-rate result.

It does not produce `ln(2)` by itself.

`ln(2)` appears only when converting binary information to entropy in natural units:

- one bit contributes `ln(2)` nats.

That is a unit-conversion fact, not a topological derivation of the cosmological constant.

So the foundation-level conclusion is:

- code rate gives a clean volume suppression law,
- but it does not close the cosmological-constant problem by itself,
- and it does not prove the Digital Horizon `ln(2)` factor as a stabilizer theorem.

## 9. Stage 5: Three Generations

### 9.1 What Is Actually Proved

The sanity script confirms the standard topological fact that the `T^3` toric code carries `3` encoded qubits.

That is consistent with `b_1(T^3) = 3`.

### 9.2 What Is Not Proved

Two additional steps are still missing:

- there is no proof that the MSQECC selection rule uniquely chooses `T^3` over the other compact flat 3-manifolds,
- and there is no proof that the three encoded qubits should be identified with the three fermion generations.

Those are extra physical hypotheses, not consequences of the present algebra.

So the three-generation claim is not yet derived.

## 10. What This Means for the Programme

The foundation test separates the framework into three layers.

### Layer A: Real and Explicit

- toric-code stabilizer algebra on `T^3`,
- encoded-qubit count `= 3`,
- code-rate scaling `= 1/L^3`.

### Layer B: Plausible but Unfinished

- using topology to motivate large-scale organizing principles,
- using morphology to extend the galaxy fit,
- treating the alpha coincidence as a target worth attacking.

### Layer C: Not Yet Derived

- `k = 3*pi`,
- inverse-square hardware noise,
- a quantitative entanglement stress tensor,
- alpha from stabilizer algebra,
- cosmological constant from code rate,
- fermion generations from code topology.

## 11. The One Honest Next Step

If MSQECC is to survive as more than phenomenology, it needs one new piece of mathematics that the plain toric code does not provide:

- an explicit matter-coupling map that tells the code how gas, stars, charge, and stress enter the Hilbert space.

Without that map:

- `k` cannot be derived,
- `T^ent` cannot be written,
- the quantum amplitude cannot be computed,
- and alpha cannot be tied to a physical `U(1)` sector.

That is the real bottleneck.

## 12. Bottom Line

Writing down the stabilizer algebra was the correct first move because it shows exactly where the theory stops.

The plain `T^3` toric code is mathematically coherent. The present MSQECC interpretation is not yet built tightly enough on top of it to produce the advertised predictions.

That is not the death of the programme. It is the end of pretending the missing bridge already exists.
