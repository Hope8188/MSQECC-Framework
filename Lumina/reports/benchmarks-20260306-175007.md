### Local Runtime Observations

- `size_vram` reported by Ollama `/api/ps`: `0`
- Loaded model: `lumina-core:latest`
- Quantization: `Q4_K_M`
- Effective runtime mode: CPU-only inference on this machine

### What This Means

The current machine is not meeting the brutal-ledger performance target, and the main reason is not just prompt verbosity. The runtime is operating with `0` VRAM offload, which strongly indicates CPU-only inference. On this hardware/runtime profile, `45 tokens/sec` is not a realistic target without a material runtime change such as GPU acceleration, a substantially smaller model, or a different inference backend.

- The benchmark harness now measures first token and machine load instead of only wall-clock latency.
